# FE → BE API 요청·검토 문서 (백엔드 api-spec)

> **대상**: 백엔드 팀  
> **목적**: 프론트가 백엔드에 보내는 **작업 요청·검토·추가 질문** 문서를 둡니다.  
> **최종 업데이트**: 2026-01-26

---

## 1. 폴더 역할

- **프론트**가 작성한 **작업 요청, 검토 요청, 추가 질문** 문서를 이 폴더에 둡니다.
- **백엔드**는 이 폴더(`docs/api-spec/`)에서 문서를 확인한 뒤 작업을 진행합니다.

---

## 2. 규칙

| 구분 | 내용 |
|------|------|
| **작성** | 프론트: 작업 요청·검토·추가 질문 문서를 여기(`docs/api-spec/`)에 작성/업로드 |
| **확인** | 백엔드: 이 폴더에서 요청 문서 확인 후 작업 수행 |
| **버전** | 추가 질문·수정이 있을 때 `문서명_v1.md`, `문서명_v2.md` 식으로 갱신 (최종 완료까지 반복) |

---

## 3. 결과 문서 위치

- 백엔드 **작업 완료 후** 결과·완료 문서는 **프론트 저장소의 `docs/api-spec/`** 에 업로드(PR, 공유 등)해 주세요.
- 프론트는 `docs/api-spec/`에서 결과를 확인한 뒤 FE 작업을 진행합니다.

---

## 4. 상세 업무 흐름

→ 프론트 문서: `docs/reference/FE_BE_API_SPEC_WORKFLOW.md` (프론트 저장소)  
→ 백엔드 문서: `docs/essentials/FE_BE_API_SPEC_WORKFLOW.md` (본 저장소)
